## 1. 角色创建最佳实践

### 1.1 角色定位清晰

**✅ 好的做法：**

```
角色名称：跨境电商客服助手
角色描述：专门处理跨境电商客户咨询的 AI 客服
核心职责：
- 回答订单相关问题
- 处理退换货咨询
- 提供物流信息查询
- 解答支付问题
```

**❌ 避免：**

```
角色名称：助手
角色描述：什么都能做的助手
```

**建议：**
- 角色名称具体明确
- 描述突出专业领域
- 职责边界清晰

### 1.2 提示词分层设计

**三层结构：**

```markdown
# 第一层：身份定义
你是一位拥有 10 年经验的资深营销顾问，
曾服务过 100+ 知名品牌，擅长数字化营销策略。

# 第二层：能力说明
你具备以下核心能力：
1. 市场分析与竞品研究
2. 营销渠道规划与优化
3. 内容营销策略制定
4. ROI 分析与预算分配

# 第三层：行为规范
- 回答要具体、可执行
- 提供数据支持和案例参考
- 如信息不足，主动提问
- 避免空泛的理论建议
```

### 1.3 场景化配置

针对不同场景创建不同角色：

| 场景 | 角色配置建议 |
|------|-------------|
| 客服咨询 | 语气友好、响应快速、标准化回答 |
| 内容创作 | 创造性高、温度 0.8+、提供多版本 |
| 数据分析 | 严谨准确、温度 0.3-、提供数据支撑 |
| 学习辅导 | 循循善诱、分步讲解、检查理解 |
| 代码开发 | 精确简洁、提供注释、考虑边界 |

### 1.4 版本管理

**迭代优化流程：**

1. **v1.0** - 基础版本，核心功能
2. **测试** - 实际使用收集反馈
3. **v1.1** - 优化提示词，修复问题
4. **A/B 测试** - 对比不同版本效果
5. **定稿** - 确定最佳版本

**版本记录：**

```markdown
## 版本历史

v2.1 (2026-02-27)
- 优化了开场白
- 增加了示例回复
- 修复了语气不一致问题

v2.0 (2026-02-20)
- 添加了行业知识库
- 调整了温度参数到 0.7
- 增加了快捷命令支持

v1.0 (2026-02-15)
- 初始版本
```

## 2. 提示词编写技巧

### 2.1 结构化提示词

**推荐结构：**

```markdown
# 角色身份
[定义 AI 的身份、背景、专业领域]

# 核心能力
[列出 AI 具备的主要能力]

# 任务流程
[描述完成任务的标准流程]

# 输出规范
[规定输出的格式、风格、长度]

# 约束条件
[说明限制和注意事项]

# 示例对话
[提供 2-3 个优质对话示例]
```

### 2.2 使用 Few-Shot 学习

**提供示例提升效果：**

```markdown
示例 1:
用户：帮我写一封产品推广邮件
助手：当然！请问：
1. 产品名称和核心功能？
2. 目标用户群体？
3. 希望突出的卖点？
4. 邮件发送场景？

示例 2:
用户：推广一款新的咖啡产品
助手：好的，这是一封产品推广邮件草稿：

主题：☕ 发现完美清晨 - [产品名] 新品上市

亲爱的咖啡爱好者，

你是否一直在寻找那一杯完美的咖啡？...
```

### 2.3 明确输出格式

**指定格式要求：**

```markdown
请按照以下格式回复：

## 核心建议
[1-2 句话总结]

## 详细分析
[分段阐述]

## 行动步骤
1. [第一步]
2. [第二步]
3. [第三步]

## 注意事项
- [注意点 1]
- [注意点 2]
```

### 2.4 设置思维链

**引导 AI 逐步思考：**

```markdown
回答问题时，请按以下步骤思考：

1. 理解问题核心 - 用户真正想要什么？
2. 分析背景信息 - 有哪些已知条件？
3. 识别关键因素 - 什么最重要？
4. 生成解决方案 - 有哪些可能方案？
5. 评估最优解 - 哪个方案最合适？
6. 组织回答 - 清晰表达结论
```

### 2.5 避免常见问题

**❌ 模糊指令：**
```
写得好一点
```

**✅ 具体指令：**
```
用专业但易懂的语言，写一份 500 字左右的产品介绍，
突出 3 个核心卖点，目标用户是 25-35 岁的都市白领
```

**❌ 多重问题：**
```
这个产品怎么样？市场前景如何？竞争对手有哪些？
应该怎么定价？推广渠道选什么？
```

**✅ 分解问题：**
```
我们先分析产品优势，再讨论市场策略。
首先，请列出这个产品的 3 个核心竞争优势。
```

## 3. 知识库管理技巧

### 3.1 文档组织策略

**按主题分类：**

```
知识库/
├── 产品文档/
│   ├── 产品手册.pdf
│   ├── 功能说明.md
│   └── 更新日志.md
├── 销售资料/
│   ├── 价格表.xlsx
│   ├── 案例研究.pdf
│   └── FAQ.md
└── 技术支持/
    ├── 安装指南.pdf
    ├── 常见问题.md
    └── 故障排查.md
```

### 3.2 文档优化技巧

**上传前预处理：**

1. **清理格式** - 移除无关样式
2. **添加标题** - 确保层次清晰
3. **补充说明** - 添加上下文信息
4. **统一术语** - 保持用词一致

**文档结构建议：**

```markdown
# 文档标题

## 概述
[简短介绍文档内容]

## 核心内容
[分章节详细阐述]

## 常见问题
[Q&A 形式]

## 相关资源
[链接到相关文档]
```

### 3.3 检索优化

**提升检索准确度：**

1. **添加元数据** - 标签、分类、关键词
2. **合理分块** - 按主题分块，避免截断
3. **设置权重** - 重要文档提高权重
4. **定期更新** - 保持内容时效性

**检索配置建议：**

| 场景 | Top-K | 相似度阈值 | 重排序 |
|------|-------|-----------|--------|
| 精确查询 | 3-5 | 0.8+ | 启用 |
| 探索查询 | 5-10 | 0.6+ | 启用 |
| 创意场景 | 10-15 | 0.5+ | 禁用 |

### 3.4 文档更新流程

**定期维护：**

- **每周** - 检查新增文档
- **每月** - 审核内容准确性
- **每季** - 清理过时文档
- **按需** - 响应用户反馈

## 4. 对话效率提升

### 4.1 提问技巧

**SCQA 框架：**

```
S (Situation) 情境：
"我正在准备产品发布会..."

C (Complication) 冲突：
"...但时间只有 2 周，资源有限..."

Q (Question) 问题：
"...如何在有限条件下最大化影响力？"

A (Answer) 答案：
[AI 给出针对性建议]
```

**5W1H 提问法：**

```
Who - 目标用户是谁？
What - 要达到什么目标？
When - 时间节点是什么？
Where - 在什么场景使用？
Why - 为什么重要？
How - 如何衡量成功？
```

### 4.2 快捷命令使用

**高效命令组合：**

```bash
# 快速总结
/总结 这篇文章的核心观点

# 深度分析
/分析 SWOT 分析这个方案

# 多语言
/翻译 成英文和日文

# 格式转换
/改写 成邮件格式

# 内容扩展
/扩展 这 3 个点，每个写 200 字
```

### 4.3 上下文管理

**有效利用上下文：**

✅ **好的做法：**
```
基于刚才的分析，请详细说明第二点的实施方案
```

❌ **避免：**
```
那个东西怎么做？（缺少明确指代）
```

**适时重置上下文：**

当话题转换时，明确说明：
```
我们换个话题，现在讨论...
```

### 4.4 多轮对话策略

**任务分解：**

```
第一轮：明确目标和约束
第二轮：头脑风暴方案
第三轮：评估和选择
第四轮：制定详细计划
第五轮：讨论执行细节
```

**进度跟踪：**

```
我们已经完成了：
✅ 问题分析
✅ 方案 brainstorm

接下来讨论：
⏳ 方案评估
⏳ 实施计划
```

## 5. 团队协作建议

### 5.1 角色共享策略

**团队角色库：**

创建团队共享角色：

1. **标准化角色** - 统一对外形象
2. **专业角色** - 各部门专用
3. **通用角色** - 全员可用

**权限管理：**

| 角色类型 | 创建 | 编辑 | 使用 | 删除 |
|---------|------|------|------|------|
| 个人角色 | ✅ | ✅ | ✅ | ✅ |
| 团队角色 | ⚠️ | ⚠️ | ✅ | ❌ |
| 公司角色 | ❌ | ❌ | ✅ | ❌ |

### 5.2 知识库协作

**知识贡献流程：**

1. **提交** - 成员上传文档
2. **审核** - 管理员审核质量
3. **分类** - 添加到合适分类
4. **发布** - 对团队可见
5. **反馈** - 收集使用反馈

**版本控制：**

- 标注文档版本

## 6. 性能优化建议

### 6.1 响应速度优化

**减少延迟：**

1. **精简提示词** - 避免冗长
2. **合理设置 Max Tokens** - 不过度生成
3. **使用快速模型** - 简单任务用 GPT-3.5
4. **预热常用角色** - 提前加载

### 6.2 成本控制

**优化 Token 使用：**

```
❌ 浪费：
请非常详细地、全面地、深入地分析这个问题...

✅ 高效：
请分析这个问题的 3 个关键点，每点 100 字
```

**模型选择策略：**

| 任务类型 | 推荐模型 | 原因 |
|---------|---------|------|
| 简单问答 | GPT-3.5 | 快速经济 |
| 复杂分析 | GPT-4 | 质量优先 |
| 长文档 | Claude | 上下文长 |
| 中文内容 | 文心一言 | 中文优化 |

### 6.3 质量保证

**质量检查清单：**

- [ ] 回答准确无误
- [ ] 格式清晰易读
- [ ] 建议可执行
- [ ] 语气符合角色
- [ ] 无敏感信息泄露

**用户反馈循环：**

1. 收集点赞/点踩
2. 分析低分对话
3. 优化提示词
4. A/B 测试改进
